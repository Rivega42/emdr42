# Правила для Claude в дизайн-задачах

> Применимо, когда Claude генерирует UI / визуальные ассеты / Figma-ready описания, а не пишет код.

## Дизайн-стек проекта

- **Tailwind CSS** — основной инструмент.
- **Framer Motion** — анимации.
- **Three.js + @react-three/fiber + drei** — 3D-сцена сессии (BLS-канвас).
- **Tone.js** — аудио (включая ASMR-паттерны).
- Палитра — нейтральные серые + brand-акценты (см. `tailwind.config.js`).
- Иконки — inline SVG, ширина по контексту, `aria-hidden` если декоративные.

## Принципы

1. **Спокойствие выше яркости.** Это терапевтическое приложение для людей в стрессе — никакого визуального шума, частых анимаций, контрастных вспышек. Особенно в `app/session/` — следим за фоторакцией (нет миганий >3 Гц).
2. **Доступность по умолчанию.**
   - Все интерактивные элементы — keyboard-navigable, с видимым `:focus-visible`.
   - `aria-label` / `aria-describedby` на иконочных кнопках.
   - Контраст ≥ 4.5:1 для текста, ≥ 3:1 для крупного.
   - Альтернатива анимации для `prefers-reduced-motion: reduce`.
3. **Адаптив.** Mobile-first. Точки слома Tailwind: `sm md lg xl 2xl`. Боковая панель `lg:translate-x-0` — переход на off-canvas ниже.
4. **Семантика.** `<header> <main> <nav> <section> <article> <aside>` вместо `<div>`-супа.
5. **Русский в UI.** Кроме служебных идентификаторов и роутов.

## Что не делать

- Не использовать утильные иконки-эмодзи в production-UI без согласования (сейчас в sidebar 🧠/🏠/📊 — это переходный mock, постепенно заменяем на SVG).
- Не злоупотреблять `motion.div` — каждое появление мотивированно (entrance, transition, feedback).
- Не вводить новые палитры без обсуждения — работаем в существующей gray-scale + accent.
- Не перестраивать существующие layout-сетки «потому что красивее».

## Если Claude передаёт дизайн в код

- Tailwind-классы в JSX **inline**, не выносить в отдельные `*.module.css`, кроме случаев сложных state-ful стилей.
- Анимации Framer — короткие декларативные пресеты (`fadeUp`, `slideIn`).
- Three.js-сцены — изолированы в `app/session/SessionCanvas.tsx` (и потомках); не подмешиваем 3D в общие layout-ы.
