# Передача задач во внешние каналы

> Когда задача уходит за пределы Claude Code (Вика, дизайнер, юрист, founders) — фиксируем её одним способом: GitHub issue + соответствующий лейбл + промт по шаблону.

## Каналы

| Получатель | Лейбл | Шаблон |
|---|---|---|
| Вика (DevOps) | `devops:vika` | [`VIKA_RULES.md`](VIKA_RULES.md) |
| Founders / Product | `статус: нужна инфа` | см. ниже |
| Юрист / Compliance | `legal` + `статус: внешний` | см. ниже |
| Дизайнер | `design` + `статус: внешний` | см. ниже |

## Шаблон для founders

```markdown
Title: [META] <вопрос>

## Контекст
<Что происходит в коде/проекте, почему вопрос возник.>

## Вопрос
<Один конкретный вопрос. Не несколько.>

## Опции
A. <вариант 1> — последствия
B. <вариант 2> — последствия
C. <вариант 3> — последствия

## Что я (Claude) рекомендую
<Опционально — одна строка.>

## Срок
До какого числа нужен ответ, чтобы не блокировать <X>.

## Связанные
<issues, PRs, документы>
```

## Шаблон для юриста

```markdown
Title: [legal] <короткое описание>

## Контекст
<Какая часть продукта затронута, что обрабатывается.>

## Запрос
<ToS / Privacy / Consent / BAA / иное>

## Юрисдикции
EU (GDPR) / RU (152-ФЗ) / US (HIPAA) / иное

## Срок
<когда нужно>

## Связанные
<issues, документы в docs/legal/>
```

## Шаблон для дизайнера

```markdown
Title: [design] <экран/компонент>

## Что нужно
<Описание, контекст использования.>

## Брэндбук / референсы
<ссылки>

## Принципы
- Доступность WCAG AA
- Спокойный визуальный язык (см. CLAUDE_DESIGN_RULES.md)
- Mobile-first

## Deliverables
- [ ] Figma frame
- [ ] States: default / hover / focus / disabled / loading / error
- [ ] Specs (spacing, typography, colors)

## Срок
```

## Правила трекинга

- Каждая внешняя задача — issue. Без issue — не существует.
- Лейбл `статус: внешний` обязателен — `validate-issue.mjs` (после ШАГА 8) может проверять.
- При получении ответа — комментарий в issue + close.
- Если ответ привёл к решению в коде — линковка `Closes #N` в PR.
