# Продуктовый контекст

> Документ для AI-исполнителей. Бизнес-видение, аудитория, метрики. Если правишь продуктовое решение — проверь, что не противоречит этому файлу. Если противоречит — обнови файл, не «обходи».

## Что мы делаем

**EMDR-AI Therapy Assistant** — цифровой помощник для проведения EMDR-сеансов (Eye Movement Desensitization and Reprocessing). Целевая ситуация: пациент работает с травмой между визитами к терапевту (или вместо первого визита, если терапевта нет рядом). AI-фасилитатор ведёт классический 8-фазный EMDR-протокол + RDI, контролируя BLS (бинауральная стимуляция движением глаз/звуком) и адаптируя ритм по голосу + эмоциям.

## Аудитория

### P1 — Терапевт-пользователь
EMDR-сертифицированный психотерапевт, ведёт пациентов и даёт им инструмент для домашней работы. Нужно:
- Назначить пациента, установить план.
- Видеть прогресс (SUDS-траектория, voice-индикаторы, safety events) между сессиями.
- Быть уведомлённым при crisis-events.
- Заметки + комментарии, видимые/скрытые от пациента.

### P2 — Пациент
Человек, работающий с травмой/тревогой/PTSD. Нужно:
- Спокойная, не-перегруженная UI.
- Безопасный пайплайн: AI не «лечит», а ведёт упражнение по протоколу + SOS-кнопка → терапевт.
- Доказательство приватности: PHI зашифровано, экспорт/удаление по запросу.

### P3 — Админ платформы
Поддерживает базу терапевтов, метрики качества, биллинг.

## Чего мы НЕ делаем (out of scope)

- Не ставим диагноз.
- Не заменяем терапевта при тяжёлой PTSD / суицидальных мыслях / диссоциации — pipeline эскалирует к человеку.
- Не работаем с детьми (<14) без отдельного flow + parental consent (в скоупе позднее).
- Не предоставляем «общий» psychology chat — только EMDR-протокол.

## Ключевые метрики (north star)

| Метрика | Цель |
|---|---|
| Среднее снижение SUDS за сессию | ≥ 2.0 |
| Завершаемость 8-фазного цикла | ≥ 60% |
| Воспринимаемая безопасность (post-session opt-in survey) | ≥ 4.5/5 |
| Crisis-events без эскалации к терапевту | < 5% |
| Сохранение пациентов 30/60/90-day | задаём после первой когорты |

## Метрики платформы

- p95 latency core voice loop (STT → LLM → TTS → BLS adapt) — < 1.2 с
- Crash-free sessions — ≥ 99%
- Backup freshness — ≤ 24 ч
- Compliance: HIPAA Technical Safeguards — passed (внешний аудит, в roadmap)

## Технологические границы

- **Voice loop** — Vosk (бесплатный fallback) + Deepgram (production); Piper + ElevenLabs.
- **LLM** — Anthropic Claude по умолчанию; OpenAI / Ollama как fallback через circuit breaker.
- **PHI** — encryption at rest (`PHI_ENCRYPTION_KEY`), encryption in transit (TLS), redaction перед LLM.

## Принципы продуктовых решений

1. **Безопасность важнее вовлечённости.** Если фича повышает retention, но снижает безопасность — отклоняем.
2. **AI — ассистент, не врач.** Все эскалационные пути ведут к живому человеку.
3. **Терапевт — со-проектировщик пути.** UX даёт ему контроль, не отбирает.
4. **Приватность пользователя — конкурентное преимущество, не compliance-обуза.**
